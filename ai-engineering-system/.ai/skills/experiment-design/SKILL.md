# Experiment Design

## Purpose

Design experiments that answer a specific question with controlled comparisons and interpretable results.

## When to use

Use this skill before implementing evaluations, ML experiments, benchmarks, ablations, or research studies.

## Decision rules

- Start with the question or hypothesis.
- Define success metrics before running experiments.
- Choose baselines that make the comparison meaningful.
- Control variables that could confound results.
- Record protocol, seeds, data splits, and analysis plan.

## Anti-patterns

- Running experiments before defining the decision they inform.
- Comparing against weak or unclear baselines.
- Changing metrics after seeing results.
- Ignoring variance, data leakage, or sample bias.

## Checklist

- Write the hypothesis.
- Define datasets and splits.
- Select baselines and metrics.
- Define controls and ablations.
- Specify logging and analysis.
- Record limitations.
