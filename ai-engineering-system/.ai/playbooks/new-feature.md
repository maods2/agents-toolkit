# Playbook: New Feature

## Use when

A new user-visible or system-visible behavior must be designed and implemented.

## References

- Principles: `principles/software-engineering.md`, `principles/architecture.md`, `principles/testing.md`
- Skills: `skills/workflow-scout/`, `skills/workflow-discovery/`, `skills/tdd/`, `skills/refactoring/`

## Workflow

1. Define the feature outcome and acceptance criteria.
2. Run Workflow Scout for the affected area.
3. Run Workflow Discovery if the area is unfamiliar or risky.
4. Identify business rules, interfaces, and impacted tests.
5. Plan the smallest useful increment.
6. Use TDD for behavior changes where practical.
7. Implement, review, and persist decisions.
