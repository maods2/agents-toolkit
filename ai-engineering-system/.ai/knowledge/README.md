# Knowledge

Store durable cross-workflow knowledge here.

Recommended files:

- `glossary.md`: shared domain terms, aliases, acronyms, and definitions.
- `business-domain.md`: durable business concepts and domain model notes.
- `architecture-principles.md`: repository-specific architecture principles and constraints.
- `adr/`: architectural decision records for significant decisions.

Discovery artifacts for a single workflow belong in `.ai/discoveries/<workflow-name>/`.
